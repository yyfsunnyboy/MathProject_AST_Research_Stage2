摘要 
針對無效刷題與雲端 AI 出題易錯之痛點，本研究旨在探討如何建構一套結合自適應題
目推薦、邊緣運算、神經符號 AI 與 RAG 檢索的自適應學習系統。首先，導入 2PL 項目
反應理論透過結合知識點相關性與不確定導向實現精準題目推薦。其次，前端藉鷹架提示卸
載認知，後端以 AST 語法樹主動修復，使邊緣 AI 出題品質接近甚至超越雲端大模型，穩
定生成程式即內容的無限題庫。最後，建構動態混合 RAG 架構，平衡向量與圖譜檢索，於
學生卡關時精準救援。本研究成功以低成本邊緣運算，實現個人化的精準教育藍圖。 
壹、 前言 
一、 研究動機 
在生成式 AI 普及的今日，許多學生雖然能夠接觸到 ChatGPT 等大型語言模型，卻往
往不知道如何有效地與這些工具互動以輔助自身學習。學生缺乏引導，難以向 LLM 提出正
確的問題和要求適合自己程度的題目，或請模型針對自己的錯誤進行根源診斷，導致強大的 
AI 工具僅淪為搜尋引擎的替代品，無法真正發揮個人化輔導的潛力。 
另一方面，現有的數位學習平台（如因材網）雖提供了結構化的練習資源，卻仍存在根
本性的痛點：題庫固定僵化、推薦機制依賴教師指派而非學生的即時能力狀態，且當學生卡
關時，缺乏根源、個人化的診斷與引導能力。學生因此陷入「無效刷題」的困境，反覆練習
已會的題目，卻在真正薄弱之處得不到精準支援。 
有鑑於此，本研究希望透過一系列實驗，分別探討自適應演算法的能力估計效能、結合
鷹架提示與 AST 修復機制的邊緣 AI 題目生成品質，以及 RAG 知識圖譜檢索在教育問答
情境中的效能，以此作為基礎，最終整合三者，設計出一套能讓學生不需具備提示工程知識、
也能享有個人化學習導航的自適應輔助學習系統，精準規劃每位學生通往精熟的最短路徑。

二、自適應學習(Adaptive Learning) 
自適應學習（Adaptive Learning, AL）[1]，旨在透過考量學生的個體差異（如學術能力、
偏好與學習進度），提供客製化且動態調整的學習路徑。相較於傳統「針對中間程度
（Teach-to-the-middle）」的教學策略，AL 能有效解決學習者能力不均的問題 。 
傳統 AL 系統多依賴機器學習進行數據分析；而生成式人工智慧的出現，則為 AL 帶來
了改變。透過 GenAI，系統能實現實時的動態題目生成與更具解釋性的回饋，不僅解決了傳
統題庫固定且僵化的問題，更能在維持統計效能的同時，顯著提升學習者的成就感與適應性。 
本研究便是以此理論為核心，運用知識圖譜的知識點依賴關係，在自適應演算法與 RAG 
智慧搜尋中提供推薦與檢索依據 。藉由 2PL 模型對能力的精準估計，搭配低成本的邊緣AI 
運算 ，針對學生學習進度診斷與精準引導，讓學生在不需具備提示工程知識的情況下，亦能
依照個人化路徑有效補強學習弱點，實現自適應學習 。

三、 項目反應理論與動態能力估計模型 
此部分文獻奠定了電腦自適應測驗（CAT）的底層數學基礎。Lord (1980) [2] 作為測量
學的經典，確立了項目反應理論（IRT），解釋如何利用數學機率推估學生的潛在能力，並提
出傳統的「最大資訊量選題邏輯」。隨著技術發展，Reckase (2009) [3]進一步將理論擴展為多
維項目反應理論（MIRT），完整整理了多科測驗所需的費雪資訊量計算與最大後驗機率（MAP）
的動態估計方法。 
本研究在能力更新與對照組設計中應用了此兩大理論。我們採用 Lord 提出的雙參數
（2PL）模型公式計算學生的答對機率，並將「最大資訊量選題法」設為本實驗的對照組。
同時，系統核心參照了 Reckase 的多維數學推導撰寫運算程式碼，確保每一次作答後，都能
精準且穩定地重新計算並定位學生的各項能力指標。 
四、 多維度知識關聯與最優化選題演算法 
此部分文獻聚焦於多學科測驗時的「關聯性應用與選題最佳化」。Segall (1996) [4]是將多
維度能力引入測驗，提出利用「共變異數矩陣」來數學化表達學科間的連動關係；而 Mulder 
等人 (2009) [5]則探討了如何在此架構下設計最聰明的選題演算法，提出透過最小化系統的
「變異數（Variance）」與「誤差矩陣的跡（Trace）」，能用最少題數讓整體的測驗誤差快速且
均勻地收斂。 
本研究在策略模組與題目推薦中使用了上述概念。首先，我們依據 Segall 的理論設定了
技能間的相關係數，讓系統具備跨知識點的連動更新能力，解決學生無效刷題的疲勞。此外，
我們將 Mulder 等人的最佳化準則轉化為程式邏輯，研發出本實驗最核心的「Variance-driven
（不確定性導向）」與「Trace-aware（圖感知）」兩種選題演算法，並於最終結果證實其選題
效能優於傳統方法。

雲端大型語言模型（如 GPT-4、Gemini Pro）具備卓越的生成品質及推理能力，但其Token

緣AI（如 Qwen-8B）成為邊緣運算的理想替代方案。然而，受限於參數量縮減，容易產生

六、 邊緣運算模型的效能提升策略 (鷹架 + 神經符號 + 修復) 
針對邊緣AI 在複雜邏輯任務中容易產生幻覺與語法崩潰的問題，學術界主要從「前端的
認知引導」與「後端的邏輯約束」兩個維度提出解決方案。本系統結合了此兩大路徑，建構
出完整的防禦體系。 
 前端預防，鷹架理論與結構化程式 
「鷹架 (Scaffolding)」概念最早由 Wood 等人 (1976) [7]提出，意指在學習者尚未具備
獨立解決問題能力時，給予暫時性的支持架構。在生成式 AI 領域，最新研究已成功將此教
育學理論轉化為結構化提示詞[8]，藉此降低模型的認知負荷。 
本研究透過預先提供標準工具（MASTER_SPEC 與 PERFECT_UTILS 工具庫），實現了
「認知卸載」，讓邊緣AI 只需專注於邏輯轉譯，從而大幅降低初始生成的錯誤率。 
 核心架構：神經符號 AI (Neuro-Symbolic AI) 
純依賴機率生成的神經網路仍無法保證 100% 的工程強健性。因此，學術界逐漸轉向神
經符號 AI (Neuro-Symbolic AI)[9]。此架構主張結合深度學習的「創造力」與傳統符號邏輯（如 
AST 抽象語法樹、Regex 規則）的「確定性」。 
在本研究中，大語言模型負責邏輯思考，而 Python 的 AST 引擎則作為符號驗證器，
形成互補。 
 後端修復：基於符號邏輯的主動修復機制 (Active Healer/APR) 
自動程式修復技術傳統多依賴 LLM 的自我反思來重寫，然而研究指出，這種自我修復
能力高度依賴模型規模，邊緣AI 往往缺乏足夠的推理能力來糾正自身錯誤，甚至容易引發二
次錯誤[10]。Khati 等人 (2026) 提出的最新研究主張，應利用 AST 進行確定性修復。 
本研究的 Active Healer 機制即是此一概念的深度實踐。

七、 基於大型語言模型的代理人與技能架構語言模型 
「推理與行動協同（ReAct）」[11] 以及「多代理人協作系統（Multi-Agent Systems, MAS）」
[12]能大幅提升系統穩定性以克服單一智能代理人（Agent）在處理複雜邏輯任務時的注意力
瓶頸。例如，MetaGPT 等研究指出，透過嚴謹的角色扮演與標準化作業流程，能有效降低程
式碼生成的錯誤率。 
本研究建構了規劃（Architect）、邊緣AI 撰寫（Coder）、自動修復（Healer）的三角協作
架構，將認知任務分流，實踐了 ReAct 框架中「思考、行動、觀察與修正」的閉環邏輯。此
外，本研究的模組化演進與Anthropic 提出的Agent skill[13]高度契合。我們將複雜的 K-12 數
學出題任務，解構為定義明確的標準化技能（整合了 SKILL.md 領域規格、結構化提示詞與 
Healer 修復機制），確保了研究的可重現性與數學教學精確度。 
八、 程式碼與數學題品質的雙軌評估指標MCRI ( Math Code Reliability Index ) 
隨著「程式即內容 (Code-as-Content)」的興起，我們不僅需要驗證 AI 生成的程式是否
強健，更需驗證其產出的內容（數學題）是否具備實際的教學價值。 
 程式碼生成評估的侷限與 L5 架構指標之建立 
CodeBLEU 與傳統指標( 如BLEU ) 相比雖引入了抽象語法樹 (AST) 與資料流分析來
改善此問題[14]，但其高度依賴標準參考答案 (Golden Template)。在數學題目動態生成的場
景具備高度多樣性，導致CodeBLEU 指標無法適用。 
為解決此痛點並順應無參考評估的趨勢 [15]，本研究在 MCRI 的「程式價值 (Program 
Value)」軌道中，借鑒解析 AST 的理念，將其升級為「無參考之靜態架構分析」。我們設計
的 L5 指標能自動掃描代碼是否具備重試機制 (Retry Loop) 與安全防護 (Safe Guard) 等防
禦性編程特徵，從根本上量化代碼的工業級強健性。 
 數學題目驗證的缺口與神經符號 (Neuro-Symbolic) 典範 
AI 生成數學題目的自動化評估，傳統方法多依賴 LLM 的自我檢查，然而多項研究證
實，現今的大語言模型在處理複雜的多步驟數學推理時，極易出現邏輯錯誤與算術失準 [14]，
例如 PAL [16]與 PoT [17]等研究皆證實，若能引導語言模型將自然語言轉化為程式碼，並將
數學計算卸載給外部直譯器執行，便能大幅提升數值運算的準確度。若讓 LLM 扮演「球員

 教學價值的量化：從認知負荷到 MQI / IS 指標 
當代研究指出，LLM 生成的數學題普遍面臨「多樣性與難度不足」的瓶頸，僅依賴 AI 
往往難以產出具備高挑戰性的題目[20]。同時，教育領域文獻強調，數位教材的品質評估必
須考量學生的認知負荷。 
本研究在 MCRI 的「數學價值 (Math Value)」軌道中，設計了 MQI (數學運算廣度) 與 
IS (推導步數深度) 兩個創新指標。IS 計算邏輯轉折的節點數（如巢狀括號與絕對值），而 MQI 
則量化了具備教學權重的運算子密度。透過這套指標，我們確保了 AI 生成的題目不只正確，
而且具備足夠的深度。 
 MCRI 雙軌評估體系 (The Dual-Track MCRI System) 
針對學界在「強健性評估」、「驗證幻覺」與「教學價值量化」的三大缺口，本研究提出
的 MCRI (Math Code Reliability Index) 建立了一套 50% 程式價值 + 50% 數學價值的雙軌
評估架構。

純向量相似度在處理具備高度階層性、邏輯性教育問題時，缺乏語意解釋力與推理深度的缺
陷。

十、 RAG（檢索增強生成） 
 Naive RAG 
Lewis 等人（2020）提出 RAG 架構[22]，結合語言模型的參數化記憶與外部知識庫，
以降低幻覺。傳統 Naive RAG 單純依賴向量相似度檢索，雖具備極速與低成本優勢，但在
處理需多步推理或缺乏直接關鍵字的複雜問題時，極易產生語意盲區。 
 Graph RAG 
為彌補 Naive RAG 的邏輯缺陷，Graph RAG 引入知識圖譜（Knowledge Graph）的結構
化特徵。透過預先萃取文本中的「核心」與「前置」概念，系統能沿著知識節點進行深度邏
輯推演，完整涵蓋知識脈絡，有效解決長尾複雜問題。 
本研究的 Hybrid RAG 架構能動態調整使用 Naive RAG 與 Graph RAG 的時機，平衡
回應速度與品質。

一、基於IRT 理論的自適應演算法研究 
 學生行為模擬：IRT 2PL 模型 
本實驗使用2PL 模型計算答對機率： 
P(θ) =
1
1 + e−a(θs−b)
D
 
此方程式是二參數邏輯模型的機率函數，計算學生以能力 θ_s 答對測量技能 s 的題目
的機率。a 是鑑別度參數，表示題目如何區分高低能力學生；b 是難度參數；D 是常數，通
常為 1.7 以近似常態分佈。此函數確保 P 在0 到1 之間，模擬二元作答（對/錯）。在多維模
型中，每題只測量一個技能 s，因此只依賴 θ_s。本實驗將IRT 模型中的能力值進行轉換。 
 實驗流程 
為了驗證上述「選題策略」的真實效能，我們進行了電腦模擬實驗：首先，在模擬環境

的建構上，我們設定了三個具備中度相關性的知識點作為測驗的底層架構。接著，系統透過
常態分布，隨機生成了 300 名帶有「隱藏真實能力」的虛擬學生，並建置了包含 120 道涵

實驗開始後，這 300 名虛擬學生將分別接受三種不同選題策略的測試。每位學生皆需作

本實驗採用「平均絕對誤差（MAE）」作為效能評估的標準。MAE 具體呈現了「系統猜測的
分數」與「學生真實分數」之間的差距，此數值越低，即代表該系統的選題策略與診斷能力
越為精準。 
 能力值更新機制：多維 MAP 能力更新 
我們透過最大後驗估計（MAP）進行能力即時更新。MAP 是系統用來「即時更新學生
能力估計」的一個核心方法。它的主要功能是：在每答完一題後，根據學生的作答結果（對
或錯），結合系統原本對學生的初步猜測，快速算出目前最合理的「能力值」。一開始，系統
對學生的能力只有一個大約的猜測（例如：這個學生可能在中上程度）。 學生答題後，系統
會看「這題的難度、鑑別力，以及學生的實際表現」，判斷原本的猜測是否合理。MAP 會同
時考慮這次作答提供的證據（資料）原本的猜測（先驗知識）然後找出一個「最平衡」的更
新值。既符合答題結果，又不會完全拋棄原本的合理猜測。 
 選題演算法設計 
 Baseline (最大 Fisher) 
本策略基於傳統自適應測驗的邏輯。系統在挑選題目時，只專注於學生當前的「單一項
技能」，並從題庫中選出在該技能上「測量精確度最高」的題目。這種做法的優點是能精準地
測出特定科目的真實程度；但缺點在於缺乏全局視野，系統容易因為過度追求某一科目的精
確度，而導致其他尚未測驗的技能被冷落，最終造成整體能力評估不均勻。 
 Variance-driven (不確定性導向) 
系統會先掃描學生所有技能的狀況，找出「估計誤差最大（即變異數最高）」的技能維度。
一旦確認哪項技能最不確定，系統就會強制從該技能中挑選題目進行測試。這是一種缺口遞

===== PAGE 11 =====
10 
補型的方式。其優先目標是「消除最不確定的部分」，確保所有技能的評估都能達到一致的準
確度，不留下診斷死角。先找出「系統目前最沒把握」的技能（也就是估計值最不確定的那
一個），然後針對這個技能挑「資訊量最高」的題目。優先解決最大的弱點。 
 Trace-aware (Graph-aware) 
本策略屬於全局優化的設計，核心在於結合了「知識關聯地圖」。系統在選題時，會預先
計算每一道題目作答後，能帶動「所有技能誤差」總和下降的幅度。透過運用知識點之間相
互影響的連動特性（例如代數好能反映一定的二元一方程式邏輯），此策略能達成「考單一題、
同步推估多科」的跨維度效益。核心想法為不只看單一技能的不確定性，選題時會選擇「整
體不確定性下降最多」的題目。 
 評估方法 
本實驗採用平均絕對誤差MAE 做為評估指標。平均絕對誤差（MAE）是用來衡量「預
測值」與「真實值」之間平均差異程度的指標。在自適應學習系統中，它代表系統所估計的
學生能力值與學生真實能力水平之間的平均偏差。其計算方法是將所有單次預測的誤差取「絕
對值」後求平均。

陸、 討論與結論 
一、不確定性與圖感知導向在資源有限的 CAT 中更高效，而圖感知需依賴技能相關性強
度。 
本實驗證實在模擬環境下多維 CAT 演算法在模擬實驗中展現顯著效能：
Variance-driven 策略精準鎖定最大不確定性技能，實現最快收斂與最低誤差（最終 MAE 
8.44，較 Baseline 下降 13.88%）；Trace-aware 策略則透過圖結構連動，在測驗中後期
進一步縮減整體不確定性，MAE 達 8.75，優於傳統最大 Fisher 資訊法（Baseline MAE 
9.80），並且在未來考慮更多變因的複雜的實驗環境中，參考更多資訊的Trace-aware 演
算法應能達到更佳的表現，甚至超越Variance-driven。或者能夠結合兩種演算法，在初期
題數較少時使用Variance-driven 並在收斂至一定程度後加入Trace-aware，以更進一步提
升表現。 
二、 鷹架提示（Scaffolding Prompt）能有效卸載認知負荷，使邊緣AI 效能大幅躍升。 
在未經輔助的 Ab1 階段，我們觀察到所有模型的數學分數偏低。我們認為，這源於 LLM 
在面對 「數學邏輯發想」 與 「程式語法轉譯」雙重任務時，遭遇了認知超載。 當引入 Ab2 
的結構化鷹架與預設工具庫後，邊緣AI 的分數與足以和大模型媲美。本系統成功將底層的重
複性運算卸載給外部 Python 函數，讓邊緣AI 得以集中在數學邏輯的編排上。這證明了在 AI 

===== PAGE 24 =====
23 
系統設計中，「給予正確的規格與工具」遠比「一味追求模型參數規模」更為關鍵。 
三、 主動修復機制能建立確定性防禦，高效彌補機率生成模型的語法隨機性。 
我們的驗證結果顯示，純依賴機率生成的語言模型，即便在鷹架引導下，仍會殘存不可
控的語法瑕疵。透過導入基於 AST 抽象語法樹的雙層修復管線（Ab3 階段），系統以確定性
的規則強制介入，精準修正了縮排與邏輯錯誤，成功將程式語法錯誤率降至零。這項機制的
介入，使得 Qwen 14B 與 8B 邊緣AI 在「程式價值」表現上微幅超越了千億參數的雲端大
模型在沒有Healer 介入的狀態。 
四、 Naive RAG 能以「零外部 API 成本」與「毫秒級極速延遲」的絕對優勢，解決 90% 的
常規數學問題。 
Hybrid C（動態混合架構）其核心價值在於精準的「題目動態分流」，展現出如同人類教
師般「簡單問題直答、複雜問題深思」的自適應（Adaptive）能力。此動態分流設計不僅大
幅節省了算力，確實彌補了純向量檢索的語意盲區。這兩份研究[23][24]均指出，單純依賴向
量相似度的檢索模型（即本研究之 Naive RAG），可能導致查詢結果與使用者需求不符，尤
其在處理具有明確邏輯結構或需跨領域語意連結的查詢任務時，其表現相對受限。當未來系
統由 50 筆測試資料擴充至動輒數萬題的海量真實題庫時，單純的向量檢索勢必會因缺乏精
確的邏輯對齊，而產生大量的「語意雜訊」。Hybrid C 結合的機制展現了將正確答案精準推
升至頂部的潛力。 
在真實應用場景，我們必須正視系統延遲與品質的取捨。儘管 Hybrid C 能夠救回難題，
但需等待圖譜推論延遲。此外，受限於目前的資料庫規模效應，當系統後端檢索深度設定為
大範圍（K=20）時，基礎的 Naive RAG 在總命中率上已逼近系統極限，使得進階演算法所
帶來的額外救援效益顯得相對微小。結論 
一、 證實該CAT 演算法在模擬環境下展現精準估計能力，MAE 低於10 分，為教育評估提供
創新方案。 
二、 主動修復機制能建立確定性防禦，有效修復機率生成模型的語法隨機性。 
三、鷹架提示（Scaffolding Prompt）能有效卸載認知負荷，使邊緣AI 效能大幅躍升。 
四、Naive RAG 能以「零外部 API 成本」與「毫秒級極速延遲」的絕對優勢，解決 90% 的
