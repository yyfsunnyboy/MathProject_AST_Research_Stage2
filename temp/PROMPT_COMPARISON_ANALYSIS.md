# Ablation Study: Prompt 对比分析

## 📊 三个版本的 Prompt 对比

### Ab1: BARE_PROMPT_TEMPLATE (基线 - 对照组)
**文件**: `ab1_bare_prompt.txt`
**大小**: ~900 字符
**特点**: 
- 自然语言，模拟普通用户的 Prompt
- 无工程化指导
- 提供 1-2 个参考例题，让 AI 自己推理
- 完全独立于 UNIVERSAL 和 MASTER_SPEC
- 无 Domain 函数库注入
- 无复杂约束规则

**测试结果**: ✅ **100 分**（成功！）

---

### Ab2: UNIVERSAL_GEN_CODE_PROMPT + MASTER_SPEC + Domain 函数库 (工程化)
**文件**: `ab2_full_prompt.txt`
**大小**: ~9000+ 字符
**特点**:
- 完整的规格书（194 行）
- 强化的约束规则（Critical rules）
- Domain 函数库自动注入
- 详细的答案格式规范
- Shuffle+Slice 安全保证
- LaTeX 格式铁律

**测试结果**: ❌ **生成失败**（失败！）

---

### Ab3: UNIVERSAL_GEN_CODE_PROMPT + MASTER_SPEC + Domain 函数库 + Healer
**文件**: 同 Ab2
**特点**:
- 同 Ab2 的 Prompt
- 额外启用 Healer 修复（Regex + AST）

**测试结果**: ⚠️ **生成成功，但答案格式仍错**（部分失败）

---

## 🎯 核心发现

| 维度 | Ab1 简潔 | Ab2 工程化 | Ab3 + Healer |
|------|--------|----------|------------|
| Prompt 长度 | ~900 | ~9000+ | ~9000+ |
| 工具库注入 | ❌ | ✅ | ✅ |
| Healer 修复 | ❌ | ❌ | ✅ |
| 约束规则数 | 2 | 20+ | 20+ |
| 生成成功 | ✅ **100%** | ❌ **0%** | ✅ **成功** |
| 答案格式正确 | ✅ | ❌ | ⚠️ **仍有问题** |

---

## 🔴 问题根因分析

### Why Ab1 wins (100分):
1. Qwen 14b 无负担，按自己最优方式生成
2. 没有「必须使用工具库」的压力
3. 没有「必须遵守 Critical 规则」的困扰
4. 自由发挥反而最好

### Why Ab2 fails:
```
Prompt 强化 (更多约束) 
  ↓
Qwen 被迫使用 Domain Functions
  ↓
但 Qwen 不会用或用错 → TypeError / AttributeError
  ↓
代码结构崩溃 ❌
```

### Why Ab3 答案格式仍错:
```
我们改 Prompt 的答案格式规则
  ↓
Qwen 看到规则后改变了生成逻辑
  ↓
但 Healer 只能修复语法，无法修复「答案逻辑」
  ↓
答案格式问题无法自动修复 ❌
```

---

## 📋 关键观察

**Ab1 Prompt 的秘诀** (900 字符搞定):
1. ✅ 简单明确的任务说明
2. ✅ 参考例题（而非规则列表）
3. ✅ 标准库，无工具库依赖
4. ✅ 自然语言，无工程术语
5. ✅ 让 Qwen 自己优化

**Ab2 Prompt 的陷阱** (9000+ 字符导致失败):
1. ❌ 过多的约束规则（20+ 条）
2. ❌ 复杂的 Domain 函数库注入
3. ❌ 强制使用特定工具
4. ❌ 详细的格式规范导致过度设计
5. ❌ Qwen 困惑于选择
