## 🔍 深入分析：為何生成這麼多函數是正常的？

### 根本原因

Bare Prompt **沒有結構約束**：
- ✅ 沒有"只生成 generate() 函數"的指令
- ✅ 沒有"使用 helper 結構"的要求
- ✅ 完全自由發揮

### Gemini 的選擇：為什麼是這個設計？

```
Gemini 分析：
"題目需要生成多個導數相關的數學問題"
    ↓
"應該有一個通用的多項式格式化器"
    ↓
"應該分別生成『極值問題』和『切線問題』"
    ↓
"最後統一在 generate(level) 中調度"

結果：最佳實踐的模塊化設計 ✅
```

### 為什麼這是好設計？

```
1. 可擴展性
   └─ 要加新題型？只需新增 generate_xxx() 函數

2. 可測試性
   └─ 可以獨立測試各個生成函數

3. 可讀性
   └─ 代碼結構清晰，維護容易

4. 復用性
   └─ format_polynomial() 被兩個生成函數共享
```
