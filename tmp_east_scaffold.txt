一、數學自動出題的典範轉移：從「純文本生成」到「程式即內容 (Code-as-Content)」 
隨著人工智慧技術的發展，電腦輔助教學中的自動出題系統經歷了從「靜態模板」到「生
成式 AI」的典範轉移 。然而，早期的文字生成模型本質上是基於機率的「下一個字預測」 。

計算錯誤。這種不穩定性使得純文字生成的 LLM 難以直接應用於對正確性要求極高的 K12

為解決此痛點，本研究導入了「程式即內容 (Code-as-Content)」的生成典範。不同於直接
要求 AI 生成題目文字，此方法主張「用 AI 撰寫程式碼來生成題目」 。參考 PAL (Program-
aided Language Models) [2]的研究概念，本系統將「邏輯推理」與「數值計算」進行分離：由
語言模型負責將數學邏輯轉化為程式結構，再由 Python 直譯器負責精確的數值運算。在邊
緣運算的架構下，此「程式即內容」模式具備四大革命性優勢： 
 根除計算幻覺 
透過將底層的計算任務卸載給 Python 直譯器與外部工具庫，徹底消除了語言模型常見
的算術錯誤，確保數學題目的絕對正確性 。 
 無限動態變體 
系統不儲存死板的靜態題目，而是儲存「能生成題目的 Python 程式碼」。不直接生成靜
態題目，而是動態生成可執行之 Python 出題腳本，實現無限變體。一支程式碼即可衍生出成
千上萬道邏輯同構、數字不同的變體題目。 
 極低邊際成本 
邊緣 AI 僅需消耗一次性的算力寫出正確的程式碼。後續的無限出題皆由本地端以近乎
零成本執行，完美解決了雲端大模型 API 調用成本昂貴且存在網路延遲的困局。 
 運算白箱化與詳解生成 (White-box Computation & Step-by-step Solutions) 
有別於傳統線上題庫「由人工撰寫程式、僅能死板替換數字」的靜態模板，本系統由 AI 
擔任「動態編程者」。在生成程式碼的過程中，AI 需調用符號運算工具庫進行逐步推導。此
「神經網路負責邏輯規劃、符號引擎負責計算」的分工架構，不僅讓運算邏輯徹底透明化，
更賦予系統自動生成「步驟化詳解」以及「針對學生迷思概念之誘答選項」的龐大潛力，真
正發揮了自適應學習的核心價值。

三、 邊緣運算模型的效能提升策略 (鷹架 + 神經符號 + 修復) 
針對邊緣AI 在複雜邏輯任務中容易產生幻覺與語法崩潰的問題，學術界主要從「前端的
認知引導」與「後端的邏輯約束」兩個維度提出解決方案。本系統結合了此兩大路徑，建構
出完整的防禦體系。 
 前端預防，鷹架理論與結構化程式 
「鷹架 (Scaffolding)」概念最早由 Wood 等人 (1976) [7]提出，意指在學習者尚未具備獨
立解決問題能力時，給予暫時性的支持架構。在生成式 AI 領域，最新研究已成功將此教育
學理論轉化為結構化提示詞[8]，藉此降低模型的認知負荷。 
本研究透過預先提供標準工具（MASTER_SPEC 與 PERFECT_UTILS 工具庫），實現了
「認知卸載」，讓邊緣AI 只需專注於邏輯轉譯，從而大幅降低初始生成的錯誤率。 
 核心架構：神經符號 AI (Neuro-Symbolic AI) 
純依賴機率生成的神經網路仍無法保證 100% 的工程強健性。因此，學術界逐漸轉向神
經符號 AI (Neuro-Symbolic AI)[9]。此架構主張結合深度學習的「創造力」與傳統符號邏輯（如 
AST 抽象語法樹、Regex 規則）的「確定性」。 
在本研究中，大語言模型負責邏輯思考，而 Python 的 AST 引擎則作為符號驗證器，形
成互補。 
 後端修復：基於符號邏輯的主動修復機制 (Active Healer/APR)

四、 基於大型語言模型的代理人與技能架構 
「推理與行動協同（ReAct）」[11] 以及「多代理人協作系統（Multi-Agent Systems, MAS）」
[12]能大幅提升系統穩定性以克服單一智能代理人（Agent）在處理複雜邏輯任務時的注意力
瓶頸。例如，MetaGPT 等研究指出，透過嚴謹的角色扮演與標準化作業流程，能有效降低程
式碼生成的錯誤率。 
本研究建構了規劃（Architect）、邊緣AI 撰寫（Coder）、自動修復（Healer）的三角協作
架構，將認知任務分流，實踐了 ReAct 框架中「思考、行動、觀察與修正」的閉環邏輯。此
外，本研究的模組化演進與Anthropic 提出的Agent skill[13]高度契合。我們將複雜的 K-12 
數學出題任務，解構為定義明確的標準化技能（整合了 SKILL.md 領域規格、結構化提示詞
與 Healer 修復機制），確保了研究的可重現性與數學教學精確度。 
五、 程式碼與數學題品質的雙軌評估指標MCRI ( Math Code Reliability Index ) 
隨著「程式即內容 (Code-as-Content)」的興起，我們不僅需要驗證 AI 生成的程式是否
強健，更需驗證其產出的內容（數學題）是否具備實際的教學價值。 
 程式碼生成評估的侷限與 L5 架構指標之建立 
CodeBLEU 與傳統指標( 如BLEU ) 相比雖引入了抽象語法樹 (AST) 與資料流分析來
改善此問題[14]，但其高度依賴標準參考答案 (Golden Template)。在數學題目動態生成的場景
具備高度多樣性，導致CodeBLEU 指標無法適用。

為解決此痛點並順應「無參考評估」的學術趨勢，本研究在 MCRI 的「程式價值」軌道
中，自行開發了基於 Python AST 模組的靜態程式碼分析器（Static Code Analyzer）作為 L5

首先過濾 ast.Call 節點以攔截 eval 等危險調用；接著分析 ast.For/ast.While 子樹中是否具備
例外處理（ast.Try），精準量化代碼是否具備「重試自癒機制 (Retry Loop)」。同時提出「程式
碼密度 (Code Density，節點數與行數比)」指標，在無需任何參考答案的前提下，從根本上以
編譯器視角量化代碼的工業級強健性。 
 數學題目驗證的缺口與神經符號 (Neuro-Symbolic) 典範 
AI 生成數學題目的自動化評估，傳統方法多依賴 LLM 的自我檢查，然而多項研究證實，
現今的大語言模型在處理複雜的多步驟數學推理時，極易出現邏輯錯誤與算術失準， PAL [16]
與 PoT [17]等研究皆證實，若能引導語言模型將自然語言轉化為程式碼，並將數學計算卸載
給外部直譯器執行，便能大幅提升數值運算的準確度。若讓 LLM 扮演「球員兼裁判」進行
自我驗證，往往無法有效挑出錯誤，引發嚴重的「計算幻覺」[18]。SymCode 論文亦指出，
純文本生成的 LLM 在數學推理極易陷入不透明的邏輯謬誤[19]。 
因此，本研究完美契合了神經符號結合 (Neuro-symbolic Approach) 的典範。僅在「題目
生成階段」拒絕使用 SymPy 進行計算，改以自定義的模組強迫模型進行分塊邏輯推導，保留
教學透明度；而在「MCRI 自動化驗證階段 (L3)」，為了克服LLM 評分數學會產生幻覺，我
們才引入SymPy 作為客觀基準，確保系統評估的絕對公平與正確。 
 教學價值的量化：從認知負荷到 MQI / IS 指標 
當代研究指出，LLM 生成的數學題普遍面臨「多樣性與難度不足」的瓶頸，僅依賴 AI 
往往難以產出具備高挑戰性的題目[20]。同時，教育領域文獻強調，數位教材的品質評估必須
考量學生的認知負荷。 
本研究在 MCRI 的「數學價值 (Math Value)」軌道中，設計了 MQI (數學運算廣度) 與 
IS (推導步數深度) 兩個創新指標。IS 計算邏輯轉折的節點數（如巢狀括號與絕對值），而 MQI 
則量化了具備教學權重的運算子密度。透過這套指標，我們確保了 AI 生成的題目不只正確，
而且具備足夠的深度。

 工具庫注入：系統預先載入函數庫。模型無需重新轉寫函式，減少底層運算錯誤。

 主動修復機制 (Active Healer) 
此為本研究之核心技術。我們將修復機制模組化為四個遞進的防護層級，將 AI 的機率
性輸出轉化為確定性程式碼，流程如圖4-3 所示。 
 
圖4-3：主動修復機制運作流程圖（圖片來源：作者繪製） 
 第一層：RegexHealer (文字級修復與防護) 
邊緣模型常洩漏中文思考過程 (Thinking Leakage)、夾雜 Markdown 標記，或生成缺乏 
break 的 while True 無窮迴圈。作為第一道防線，系統透過正規表達式 (Regex) 強制剔除不
合法的 Markdown 標籤與跳脫字元。更關鍵的是，此層實作了「迴圈熔斷 (Loop Breaker)」，
一旦掃描到無窮迴圈特徵，即強制注入計數器，從根本防止沙盒執行時的系統當機。 
 第二層：ASTHealer (語法樹級修復) 
模型常因縮排錯誤導致 IndentationError，或在函式末尾遺漏 return 語句，導致程式結構
破裂。系統將初步清理後的代碼解析為抽象語法樹 (Abstract Syntax Tree, AST)。有別於傳統
字串替換，系統能在語法樹層級精準定位錯誤節點，自動重構錯誤的縮排層級，並於特定函
式節點（ast.FunctionDef）尾端強制補全缺失的 return 語句。 
 


===== PAGE 11 =====
10 
 AntiDuplicationHealer (去重與變數修復) 
邊緣 AI 易產生「知識衝突幻覺」，例如重複宣告相同變數、變數使用順序顛倒，甚至發
生變數遮蔽 (Variable Shadowing)（覆蓋底層預定義的工具函數）。透過靜態變數追蹤技術，系
統會移除重複宣告的變數，並重新排序賦值邏輯，確保資料流 (Data Flow) 的邏輯自洽性。同
時，系統會將 AI 捏造的幻覺函數強制重定向至我們預設的標準工具庫 API。 
 UnifiedCleanupHealer 與動態語意回饋 
邊緣 AI 生成的程式中，常殘留未使用的變數佔用記憶體，或潛藏深層的動態執行錯誤
（RuntimeError）。系統首先執行「單次 AST 遍歷 (Single Pass Scan)」，一次性清除未使用的
變數 (Dead Code) 並進行最終的 PEP8 代碼格式化。隨後，將此潔淨代碼 (Clean Code) 送入
隔離沙盒執行。若執行時仍觸發例外錯誤，系統將擷取錯誤堆疊 (Exception Traceback) 啟動
動態回饋機制，交由大語言模型進行深層的語意重寫 (Semantic Heal)。 
 資訊科學技術貢獻小結 
整體而言，本研究之 Active Healer 修復管線並非單純的提示詞工程，而是深度整合了多
項核心的資訊科學技術，包含：詞法分析與正規表達式處理 (Lexical Analysis & Regex)、抽象
語法樹節點解析與變換 (AST Parsing and Node Transformation)、作用域與變數遮蔽檢測 
(Scope & Variable Shadowing Detection) ，以及動態例外堆疊分析與沙盒執行 (Dynamic 
Exception Traceback & Sandbox Execution)。此機制的建立，具體展現了神經網路與符號邏輯
結合的強大防禦力。

陸、 討論與結論 
一、鷹架提示（Scaffolding Prompt）能有效卸載認知負荷，使邊緣AI 效能大幅躍升。 
在未經輔助的 Ab1 階段，我們觀察到所有模型的數學分數偏低。我們認為，這源於邊緣
AI 在面對 「數學邏輯發想」 與 「程式語法轉譯」雙重任務時，遭遇了認知超載。當引入 
Ab2 的結構化鷹架與預設工具庫後，邊緣AI 的分數與足以和大模型媲美。本系統成功將底
層的重複性運算卸載給外部 Python 函數，讓邊緣AI 得以集中在數學邏輯的編排上。這證明
了在 AI 系統設計中，「給予正確的規格與工具」遠比「一味追求模型參數規模」更為關鍵。 
二、 主動修復機制能建立確定性防禦，高效彌補機率生成模型的語法隨機性。 
我們的驗證結果顯示，純依賴機率生成的語言模型，即便在鷹架引導下，仍會殘存不可
控的語法瑕疵。透過導入基於 AST 抽象語法樹的雙層修復管線（Ab3 階段），系統以確定性
的規則強制介入，精準修正了縮排與邏輯錯誤，成功將程式語法錯誤率降至零。這項機制的
介入，使得 Qwen 14B 與 8B 邊緣AI 在「程式價值」表現上微幅超越了千億參數的雲端大
模型在沒有Healer 介入的狀態。 
三、 模組化離線驗證之必要性與專家效度 
針對本系統目前「尚未於真實課堂進行端對端部署測試」之限制，本研究採取「先模組
化離線驗證，後端對端整合」之嚴謹策略。考量到 K12 教育的正確性不容妥協，若未體驗證
