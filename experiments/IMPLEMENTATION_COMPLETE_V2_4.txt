【修改 1】定義工具庫手冊常數 (Lines 210-256)
────────────────────────────────────────────

新增 4 個工具手冊定義在 PromptBuilder 類別中：

  • MANUAL_INTEGER: 基礎與整數工具
    └─ fmt_num(), random_nonzero(), is_divisible()
  
  • MANUAL_FRACTION: 分數工具
    └─ create(), to_latex(), add/sub/mul/div()
  
  • MANUAL_RADICAL: 根號與幾何工具
    └─ create(), to_latex(), is_perfect_square()
  
  • MANUAL_CALCULUS: 微積分與多項式工具
    └─ create_poly(), differentiate(), to_latex()


【修改 2】實作動態選擇方法 (Lines 260-320)
────────────────────────────────────────────

新增 2 個方法到 PromptBuilder 類別：

  1. _get_dynamic_api_manual(skill_name, skill_desc) -> tuple
     ├─ 掃描關鍵字
     ├─ 根據檢測結果組裝工具手冊
     └─ 返回: (manual_text, active_tools_list)
  
  2. _build_tool_selection_protocol(active_tools) -> str
     └─ 構建工具選用協定
        (強制 LLM 正確使用工具)


【修改 3】更新 build() 方法 (Lines 325-415)
────────────────────────────────────────────

修改 build() 方法以支持動態工具選用：

  • Ab1 模式: 無變更 (BARE_PROMPT_TEMPLATE 不含動態工具)
  
  • Ab2 模式: 新增動態工具選用 ✨
    ├─ 調用 _get_dynamic_api_manual()
    ├─ 調用 _build_tool_selection_protocol()
    └─ 將兩者注入 Prompt
  
  • Ab3 模式: 新增動態工具選用 ✨
    └─ 同 Ab2

具體變更：
  - 行 364-370: 添加 skill_id 的動態檢測邏輯
  - 行 348-355: Ab2 模式的動態工具選用代碼
  - 行 374-381: Ab3 模式的動態工具選用代碼

【使用示例】
================================================================================

基本用法：
```python
from core.prompts.prompt_builder import PromptBuilder

# 自動檢測並組裝工具手冊
prompt = PromptBuilder.build(
    master_spec=your_spec,
    ablation_id=2,
    topic="分數的四則運算",
    skill_id="jh_數學1上_FourArithmeticOperationsOfNumbers"
)
# ✅ 系統自動檢測「分數」關鍵字
# ✅ 自動啟用 FractionOps
# ✅ 植入分數運算的工具協定
```

進階用法：
```python
# 手動檢查啟用的工具
manual, tools = PromptBuilder._get_dynamic_api_manual(
    "分數運算",
    "學習分數的加減乘除"
)
print(tools)  # ['IntegerOps', 'FractionOps']
```
