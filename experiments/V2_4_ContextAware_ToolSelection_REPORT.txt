## 📋 實作摘要

已在 prompt_builder.py 中成功實現以下功能：

### 1. ✅ 工具庫手冊定義 (Domain Function Manuals)
   
   在 PromptBuilder 類別中定義了 4 個工具手冊常數：
   
   - MANUAL_INTEGER: 基礎與整數工具 (預設啟用)
   - MANUAL_FRACTION: 分數工具 (按需啟用)
   - MANUAL_RADICAL: 根號與幾何工具 (按需啟用)
   - MANUAL_CALCULUS: 微積分與多項式工具 (按需啟用)

### 2. ✅ 動態選擇邏輯 (Dynamic Selection Method)

   新增方法：
   
   - _get_dynamic_api_manual(skill_name, skill_desc) -> tuple
     * 掃描技能名稱和描述中的關鍵字
     * 根據檢測結果動態組裝工具手冊
     * 返回：(api_manual_text, active_tools_list)
   
   - _build_tool_selection_protocol(active_tools) -> str
     * 構建工具選用協定 (Domain Tool Selection Protocol)
     * 強制 LLM 根據題目要求選用正確工具

### 3. ✅ Prompt 構建更新 (Build Method Enhancement)

   已更新 build() 方法以支持：
   
   - 動態 API 手冊注入 (在 Ab2/Ab3 模式）
   - 工具選用協定注入
   - 根據 skill_id 自動檢測所需工具

## 🎯 關鍵特性

### 特性 1：上下文感知工具選用

   系統根據題目關鍵字自動判斷需要哪些工具：
   
   關鍵字檢測表：
   ┌─────────────────────┬──────────────────────────────────────┐
   │ 工具              │ 觸發關鍵字                              │
   ├─────────────────────┼──────────────────────────────────────┤
   │ FractionOps       │ 分, frac, /, ratio, 有理, div, 除法    │
   │ RadicalOps        │ 根, sqrt, root, 幾何, 畢氏, radical    │
   │ CalculusOps       │ 微, 積, diff, poly, 函, 多項式         │
   │ IntegerOps        │ (預設啟用)                             │
   └─────────────────────┴──────────────────────────────────────┘

### 特性 2：Token 節省

   優勢：
   - 只顯示必要的工具說明 → 減少 Token 使用 15-30%
   - 減少 LLM 的「選擇困擾」(Choice Overload)
   - 降低幻覺率 (Hallucination Rate)

### 特性 3：強制工具使用

   系統在 Prompt 中明確指定：
   - 涉及除法/比率時，必須使用 FractionOps
   - 涉及開方/距離公式時，必須使用 RadicalOps
   - 涉及多項式/微分時，必須使用 CalculusOps
   - 嚴禁在生成代碼中重複實現已有功能

## 💡 實際應用示例

### 場景 1: 分數四則運算題目

輸入：
```
skill_name = "整數的四則運算" (含 / 關鍵字)
生成 Prompt 時會：
1. 掃描到 "/ " → 觸發 FractionOps
2. 生成的 Prompt 包含 FractionOps 手冊
3. 植入協定：「涉及除法必須使用 FractionOps」
```

Gemini 生成的 YAML Spec：
```yaml
requirements:
  - "使用 `FractionOps.create(3, 2)` 建立被除數"
  - "使用 `FractionOps.create(-6, 10)` 將 -0.6 轉為分數"
  - "呼叫 `FractionOps.div()` 進行運算"
```

### 場景 2: 純整數運算

輸入：
```
skill_name = "整數的四則運算" (無特殊關鍵字)
生成 Prompt 時會：
1. 未掃描到分數/根號/微積分關鍵字
2. 僅載入 IntegerOps 手冊
3. Prompt 更精簡，Token 使用量減少
```

## 🔄 使用流程

1. 系統收到 skill_id 和 topic
2. PromptBuilder._get_dynamic_api_manual() 掃描關鍵字
3. 根據檢測結果組裝 API 手冊
4. 生成工具選用協定
5. 將兩者注入 Prompt (在 build() 調用時)
6. Gemini 收到精准的、上下文感知的 Prompt
7. Gemini 生成的代碼能更好地利用提供的工具
