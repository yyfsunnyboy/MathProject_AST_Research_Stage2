## 🔬 從「動態提示」到「程式即內容」的典範轉移

### 1. 動態生成的問題診斷

#### 原有設計
```
每次實驗 → 動態組裝 Prompt → AI 生成代碼 → 評估結果
```

#### 發現的問題

| 問題 | 影響 | 嚴重性 |
|------|------|--------|
| **Prompt 內容不固定** | 無法確保實驗可重現 | 🔴 高 |
| **Ab2/Ab3 可能使用不同 Prompt** | 無法純粹比較 Healer 效果 | 🔴 高 |
| **動態生成耗費時間** | 批次實驗效率低下 | 🟡 中 |
| **研究焦點模糊** | 看起來在展示 PromptBuilder | 🟡 中 |

**關鍵洞察**：
- 動態生成會引入**不可控的隨機性**，導致生成品質不穩
- 無法區分「Prompt 變化」和「Healer 效果」的各自貢獻
- 違反了科學實驗「控制變因」的基本原則

### 2. Golden Prompt 的解決方案

#### 新設計
```
一次性準備 → 固定 Golden Prompt → 批次實驗 → 可重現結果
```

#### 核心改進

**標準化 (Standardization)**：
- 將 Prompt 固定為高品質的「黃金範本 (Golden Prompt)」
- 確保所有實驗使用完全相同的模型輸入
- Ab2/Ab3 共用同一個 Golden Prompt（差異僅在 Healer 開關）

**動態內化 (Internalized Dynamism)**：
- 將「變化的需求」轉移至 Python 程式碼的 random 邏輯中
- 而非 Prompt 文字本身
- AI 生成的是「出題引擎」，而非「具體題目」

**角色轉變 (Role Transformation)**：
```
從：每次都要重新創作的「作家」（高風險）
到：設計自動化模具的「工程師」（高穩定）
```

## 🛠️ 技術實作

### 架構演進

#### 階段 1：動態生成時代（2025-2026.01）
```python
# 每次都重新生成 Prompt
def generate_code(skill_id, ablation_id):
    prompt = PromptBuilder.build(skill_id, ablation_id)  # ❌ 每次不同
    code = ai_generate(prompt)
    return code
```

**問題**：
- Prompt 可能包含隨機元素（例如範例選擇）
- 無法保證 Ab2 和 Ab3 使用相同的 Prompt
- 實驗結果難以重現

#### 階段 2：Golden Prompt 時代（2026.02.04 起）
```python
# 模式 4：準備階段（一次性）
def mode_4_prepare(skill_id):
    # 生成並保存 Golden Prompt
    ab1_prompt = PromptBuilder.build(skill_id, ablation_id=1)
    save_to_file(f'{skill_id}_Ab1.txt', ab1_prompt)
    
    ab2_prompt = PromptBuilder.build(skill_id, ablation_id=2)
    save_to_file(f'{skill_id}_Ab2.txt', ab2_prompt)  # ✅ Ab2/Ab3 共用

# 模式 2：實驗階段（批次執行）
def mode_2_experiment(skill_id, ablation_id):
    # 讀取固定的 Golden Prompt
    if ablation_id == 1:
        prompt = load_from_file(f'{skill_id}_Ab1.txt')
    else:
        prompt = load_from_file(f'{skill_id}_Ab2.txt')  # ✅ Ab2/Ab3 相同
    
    code = ai_generate(prompt)
    return code
```

**優勢**：
- ✅ 輸入固定 → 實驗可重現
- ✅ Ab2/Ab3 使用完全相同的 Prompt → 單一變因
- ✅ 批次生成效率提升 50%+

### 實驗設計對比

#### 舊設計（多變因）
```
Ab2: [動態 Prompt] → AI 生成 → [無 Healer] → 結果
Ab3: [可能不同 Prompt] → AI 生成 → [有 Healer] → 結果

問題：無法確定差異來自 Prompt 還是 Healer
```

#### 新設計（單變因）✅
```
Ab2: [固定 Golden Prompt] → AI 生成 → [無 Healer] → 結果
Ab3: [相同 Golden Prompt] → AI 生成 → [有 Healer] → 結果

清晰：差異完全來自 Healer（單一變因）
```

## 📈 實驗方法論提升

### 1. 可重現性 (Reproducibility)

**問題**：
- 動態生成導致每次實驗輸入不同
- 其他研究者無法驗證結果

**解決**：
- Golden Prompts 可納入版本控制（Git）
- 任何人使用相同 Prompt 都能重現實驗

### 2. 變因控制 (Control Variables)

**問題**：
- Prompt 變化 + Healer 開關 = 兩個變因混雜
- 無法確定哪個因素影響結果

**解決**：
- 固定 Prompt → 唯一變因是 Healer
- 符合科學實驗「單一變因」原則

### 3. 統計有效性 (Statistical Validity)

**問題**：
- 輸入不固定導致方差過大
- 難以達成統計顯著性

**解決**：
- 相同輸入 → 減少隨機誤差
- 提升統計檢定的效力
