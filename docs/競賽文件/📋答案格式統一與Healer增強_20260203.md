### **方案 B：Healer 層級增強（Layer 2）**

**修改檔案**: `core/healers/regex_healer.py`

**新增規則**: Rule 2B（空格分隔符修復）

位置：在現有 Rule 2（逗號分隔修復）之後添加

**功能說明**:
```python
# Rule 2 (existing): 逗號分隔修復
# 檢測並修復: ', '.join(answers) → '\n'.join(answers)

# Rule 2B (NEW): 空格分隔修復  
# 檢測並修復:
#   - ' '.join(answers) → '\n'.join(answers)
#   - " ".join(answers) → '\n'.join(answers)
#   - \s+ 模式的空格分隔
```

**實現詳節**（行號 199-230）:
```python
def _fix_answer_format(self, code):
    """修復答案格式 (Rule 2 + 2B)"""
    
    # Rule 2: 逗號分隔 (existing)
    pattern_comma = r"(['\"])([,])['\"]\.join\((\w+)\)"
    if re.search(pattern_comma, code):
        code = re.sub(pattern_comma, r"'\n'.join(\3)", code)
        self.healer_count['fix_answer_format'] = 1
    
    # Rule 2B: 空格分隔 (NEW)
    # 匹配 ' '.join, " ".join, \s+ 等空格模式
    patterns_space = [
        r"[\'\"][\s]+[\'\"]\.join\(",  # ' '.join, " ".join, '  '.join 等
        r"[\'\"][\s\t]+[\'\"]\.join\(",  # 包含 tab
    ]
    
    for pattern in patterns_space:
        if re.search(pattern, code):
            # 將所有空格分隔替換為換行分隔
            code = re.sub(
                pattern,
                r"'\n'.join(",
                code
            )
            self.healer_count['fix_answer_format'] = (
                self.healer_count.get('fix_answer_format', 0) + 1
            )
    
    return code
```

## 📝 系統設計要點

### **為什麼需要雙重防護**

1. **Layer 1 不夠**：
   - Prompt 最佳實踐無法完全控制 LLM 行為
   - 文檔矛盾會對未來生成造成困擾
   - 預防性修複成本最低

2. **Layer 2 不夠**：
   - Healer 的修復規則有限
   - 新型錯誤可能出現（如本次發現的空格分隔）
   - 需要持續擴展 Healer 規則庫

3. **雙重防護**：
   - Layer 1 杜絕根源（指導清晰）
   - Layer 2 兜底防禦（自動修復）
   - 形成「進攻 + 防守」的完整體系

### **Healer 的進化方向**

本次增強展示了 Healer 的擴展性：


### **Prompt 工程的啟示**

這個案例揭示了：

1. **多層 Prompt 的風險**：
   - 當同一系統有多個 Prompt 層級時，矛盾很容易出現
   - UNIVERSAL_PROMPT + MASTER_SPEC 衝突（詳見另一文檔）
   - 答案格式指導的 5 處矛盾

2. **文檔驅動開發的必要性**：
   - 所有假設和規格都應該明確文檔化
   - 定期審計文檔一致性
   - 使用工具（grep、regex 搜尋）自動檢測矛盾

3. **防禦性編程**：
   - 無論 Prompt 多清晰，都要假設 AI 會出錯
   - 主動構建 Healer 機制而非被動補救
   - 讓系統具備「自癒能力」

   - 自動修復機制是必備，不是可選

2. **小模型 + 強 Healer = 大模型**：
   - 本案例證明：14B + 完整 Healer ≈ Gemini Pro
   - 成本降低 90%+，穩定性相當或更好
   - 「賦能小模型」的策略完全可行
