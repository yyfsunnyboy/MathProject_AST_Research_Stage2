## 🔧 技術實作

### 1. Golden Prompt 讀取功能（模式 2）

**文件**：`core/code_generator.py`, `scripts/sync_skills_files.py`

**功能**：
- 模式 2 讀取固定的 Golden Prompt 文件
- Ab1 讀取：`{skill_id}_Ab1.txt`
- Ab2/Ab3 共用：`{skill_id}_Ab2.txt`（關鍵設計！）
- Fallback 機制：文件不存在時降級為動態生成

### 2. Golden Prompt 自動保存功能（模式 4）

**文件**：`core/code_generator.py`, `scripts/sync_skills_files.py`

**功能**：
- 模式 4 在動態生成 Prompt 時自動保存為 Golden Prompt
- Ab1 保存到：`{skill_id}_Ab1.txt`
- Ab2/Ab3 保存到：`{skill_id}_Ab2.txt`（共用同一文件）
- 自動創建目錄結構

**測試結果** ✅：

✅ 保存內容與生成內容一致

【測試 3】Ab3 保存到 Ab2 文件
   💾 已保存 Golden Prompt: jh_LinearEquations_Ab2.txt

## 🔬 實驗設計改進

### 變因控制對比

#### 舊設計（多變因）
```
Ab2: [動態 Prompt] → AI 生成 → [無 Healer] → 結果
Ab3: [可能不同 Prompt] → AI 生成 → [有 Healer] → 結果

問題：
- Prompt 可能不同 → 混淆變因
- 無法純粹比較 Healer 效果
```

#### 新設計（單一變因）✅
```
Ab2: [固定 Golden Prompt] → AI 生成 → [無 Healer] → 結果
Ab3: [相同 Golden Prompt] → AI 生成 → [有 Healer] → 結果

優勢：
- Prompt 完全相同 → 單一變因
- 差異純粹來自 Healer → 因果明確
```

### 科學價值提升

| 維度 | 提升 |
|------|------|
| 可重現性 | 低 → 高 |
| 變因控制 | 弱 → 強 |
| 因果推論 | 模糊 → 清晰 |
| 統計效力 | 低 → 高 |
| 論文品質 | 應用 → 研究 |
