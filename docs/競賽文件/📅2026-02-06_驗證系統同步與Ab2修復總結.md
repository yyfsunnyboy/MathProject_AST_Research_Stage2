### 問題 1: 驗證系統不同步
**症狀**:
- `run_experiment.py` 和 `sync_skills_files.py` 使用不同的驗證邏輯
- 同一份 Ab2.txt Prompt：
  - ✅ `sync_skills_files.py` → 生成成功 (Ab2.py)
  - ❌ `run_experiment.py` → 驗證失敗 (Ab2_run01.py)

**根本原因**:
`verify_code_integrity()` 依賴粗糙的括號計數，不適用於包含字符串和註解的代碼
```python
# 舊邏輯 (V47.7):
open_braces = code_str.count('{')     # 計字符串中的 { 
close_braces = code_str.count('}')    # 計字符串中的 }
if open_braces != close_braces:
    return "FAILED"  # ❌ 誤判！LaTeX 格式字符串有 {...}
```

**問題舉例**:
- Ab2_run01.py 中有 `\frac{{{rem_num}}}{{{abs_num.denominator}}}`
- 計數結果: `{ 83 vs } 85` → 誤判為代碼被截斷

### 問題 2: Ab2.py 生成邏輯缺陷
**症狀**:
- LLM 生成的 Ab2.py 在第 605 行有 `continue` SyntaxError
- 複雜的括號驗證邏輯導致無法通過測試

**修復方案**:
手動重寫整個 while True 迴圈邏輯，改為清晰的「逐步驗證」結構：
```python
while True:
    # 生成操作數與運算符
    # 驗證至少有乘除法
    if not any(op in ['*', '/'] for op in operators_A):
        continue  # ← 清晰的 continue 語句
    
    # 逐步計算，每步檢查整除性和範圍
    temp_val = operands_A[0]
    for i, op in enumerate(operators_A):
        if op == '/' and temp_val % operands_A[i+1] != 0:
            valid = False
            break  # ← 正確的控制流
    
    # 所有驗證通過才 break
    if valid:
        break  # ← 唯一的正常退出點
```

```python
def verify_code_integrity(code_str):
    """
    完整代碼驗證流程
    1. 檢查結構 (generate, check 函數存在)
    2. 編譯檢查 (compile() → Python 最準確的語法驗證)
    3. 動態採樣 (執行 generate() 檢查輸出格式)
    """
    # ✅ 移除粗糙的括號計數
    # ✅ 依賴 compile() 作為主要驗證機制
    # ✅ 動態執行 generate() 檢驗實際功能
    
    # Step 1: 結構檢查
    if 'def generate(' not in code_str:
        return "FAILED"
    if 'def check(' not in code_str:
        return "FAILED"
    
    # Step 2: 編譯檢查
    try:
        compile(code_str, '<string>', 'exec')
    except SyntaxError:
        return "FAILED"
    
    # Step 3: 動態採樣 (執行 1 次)
    try:
        namespace = {}
        exec(code_str, namespace)
        generate_func = namespace['generate']
        result = generate_func()
        
        if not isinstance(result, dict):
            return "FAILED"
        if 'question_text' not in result or 'answer' not in result:
            return "FAILED"
        
        return "PASSED"
    except Exception:
        return "FAILED"
```
