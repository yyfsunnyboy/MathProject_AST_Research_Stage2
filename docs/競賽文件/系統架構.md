#### 防禦層 2️⃣：提示層 - Golden Prompt 反 C 語言指令

✅ **新增指引**: ⚠️ PYTHON SYNTAX STRICTNESS 部分
- **位置**: Ab2 Golden Prompt 中
- **內容**: 5 條明確規則 + 警告信息
  ```
  1. ❌ NEVER put a closing brace `}` at the end of the script.
  2. ❌ NEVER use `return { ... };` (semi-colon).
  3. ❌ NEVER use C-style syntax like `};` to end blocks.
  4. ✅ Just end the `def generate` function naturally with indentation.
  5. ✅ Python dictionaries use `{ ... }` INSIDE expressions, not at block level.
  ```
- **效果**: 從源頭減少 14B 模型生成 C-style 代碼概率

#### heal() 方法流程更新

```

  Step 1            → remove_markdown_fences()          (Markdown 標籤移除)

  Step 3            → fix_common_syntax_errors()        (語法修復)
  Step 4            → remove_input_calls()              (input 移除)
  
返回：
  - fixed_code: 修復後代碼
  - stats_dict: {'regex_fix_count': N, 'markdown_removed': T/F, ...}
```

## 🆕 IX. experiment_runs 新增欄位詳解 (2026-02-05)

### **群組 1: 批次管理 (Batch Management)**

**目的**: 讓你一次撈出「某次跑的 50 筆數據」畫圖才不會亂

- SQL 查詢: `SELECT * FROM experiment_runs WHERE batch_id='exp_20260205_001'`
- 用於生成批次統計報告與對比分析

---

### **群組 2: Golden Prompt 變因控制 (Control Variables)**

**目的**: 證明 Ab2 (Engineered) 和 Ab3 (Healer) 用的是完全一樣的 Prompt

| 欄位 | 類型 | 說明 | 範例 |
|-----|------|------|------|
| golden_prompt_path | VARCHAR(255) | Golden Prompt 文件完整路徑 | 'experiments/golden_prompts/temp/jh_LinearEquations_Ab2.txt' |
| prompt_hash | VARCHAR(64) | Prompt 的 SHA256 雜湊值 | 'a7b2f4c1d8e9f0...' |

**用途**:
- **變因控制**: 確保 Ab2 和 Ab3 的 Prompt 完全相同（只有 Healer 開關不同）
- **可重現性**: 任何人都可驗證使用的確切 Prompt 版本
- **版本追蹤**: 如果 Prompt 變更，hash 值立即發現
- SQL 查詢: `SELECT DISTINCT prompt_hash FROM experiment_runs WHERE ablation_id IN (2, 3) AND skill_name='jh_LinearEquations'`

---

### **群組 3: 成本與效能指標 (Cost & Performance)**

**目的**: 計算 CP 值，證明 Healer 多花的 Token 是值得的

| 欄位 | 類型 | 說明 | 用途 |
|-----|------|------|------|
| prompt_tokens | INTEGER | Prompt 部分的 Token 數 | 成本分析、與 Ab2 對比 |
| completion_tokens | INTEGER | 模型生成部分的 Token 數 | 成本分析 |
| total_tokens | INTEGER | 總 Token 數 (Prompt + Completion) | API 計費基礎 |
| latency_ms | INTEGER | 整個流程的延遲（毫秒） | 性能分析 |

**計算範例**:
```
成本差異 = (Ab3_tokens - Ab2_tokens) / Ab2_tokens * 100%
性能提升 = (Ab2_success - Ab1_success) / Ab1_success * 100%
CP 值 = 性能提升 / 成本差異
```

**統計查詢**:
```sql
SELECT 
    ablation_id,
    AVG(total_tokens) as avg_tokens,
    AVG(latency_ms) as avg_latency,
    COUNT(*) as sample_count
FROM experiment_runs
WHERE skill_name='jh_LinearEquations'
GROUP BY ablation_id;
```

---

### **群組 4: Healer 介入統計 (Healer Metrics)**

**目的**: 量化證明「Healer 真的有在工作」，而不只是掛名

| 欄位 | 類型 | 說明 | 值域 |
|-----|------|------|------|
| healer_applied | BOOLEAN | 是否啟用 Healer | 0=Ab2 (無), 1=Ab3 (有) |
| healer_fix_count | INTEGER | Healer 修復的次數 | 0~9 (對應 9 層修復) |

**Healer 修復層級對應**:
```
healer_fix_count = N 表示修復應用了 N 層機制：
1. Whitespace Fix
2. Import Cleanup
3. Function Wrapping
4. Anti-Override
5. Mojibake Removal
6. Regex Healer
7. AST Healer
8. Eval Eliminator
9. Loop Breaker
```

**統計查詢**:
```sql
SELECT 
    healer_applied,
    AVG(healer_fix_count) as avg_fixes,
    COUNT(*) as total_runs,
    SUM(CASE WHEN avg_mcri_total > 80 THEN 1 ELSE 0 END) as success_count
FROM experiment_runs
WHERE skill_name='jh_LinearEquations'
GROUP BY healer_applied;
```

**預期結果**:
- Ab2 (healer_applied=0): avg_fixes=0
- Ab3 (healer_applied=1): avg_fixes=2.5 (平均修復 2~3 層)

## 🎯 V. 核心技術架構

### Ablation Study 三層次

| 組別 | 代稱 | Prompt | Healer | 目的 |
|------|------|--------|--------|------|

| **Ab3** | Full | UNIVERSAL_GEN_CODE_PROMPT | ✅ 完整 | Healer 修復效果 |

### Healer 系統架構

**9 層修復機制**（按順序執行）:

```
1. Whitespace Fix → 縮排修正
2. Import Cleanup → 重複導入清理
3. Function Wrapping → 函數保護包裝
4. Anti-Override → 防系統函數重複定義
5. Mojibake Removal → UTF-8 亂碼註解清除 ✨
6. Regex Healer → 正則表達式修復
7. AST Healer → 語法樹修復
8. Eval Eliminator → 移除危險的 eval 調用
9. Loop Breaker → 偵測並防止無限迴圈
```

### MCRI 評分系統（V4.2.1）

**四層級評估架構**:

```
L1 層 (語法檢驗) - 40 分
├─ 語法有效性 (20 分)
│  └─ AST 可解析，無語法錯誤
└─ 導入有效性 (20 分)
   └─ 所有導入都可解析

L2 層 (邏輯檢驗) - 50 分
├─ 邏輯正確性 (25 分)
│  └─ 符合教學期望
├─ 能否執行 (15 分)
│  └─ Runtime 無崩潰
└─ 執行結果 (10 分)
   └─ 輸出格式正確

L3 層 (一致性檢驗) - 30 分
├─ 內在一致性 (15 分)
│  └─ Check(ans, ans) == True
└─ 外在強健性 (15 分)
   └─ 對不同輸入反應穩健

L4 層 (教育價值) - 20 分
└─ 教學品質
   ├─ 邏輯清晰度
   ├─ 代碼可讀性
   └─ 教學有效性

總分: 0~100 分
```

### 📍 第一站：定義工具 (Define)

**檔案**: `core/prompts/domain_function_library.py`

這是您的「軍火庫」，先把武器造出來。

```python
# ... (原本的代碼) ...

# ===== [新增] 5. 幾何運算模組 (GeometryOps) =====
class GeometryOps:
    """
    幾何運算專用工具
    """
    
    @staticmethod
    def triangle_area(base, height):
        """
        計算三角形面積 (回傳 Fraction)
        例: triangle_area(3, 5) → Fraction(15, 2)
        """
        # 這裡可以呼叫 FractionOps 確保精度
        return FractionOps.mul(FractionOps.mul(base, height), FractionOps.create(Fraction(1, 2)))

    @staticmethod
    def circle_area(radius, pi_symbol="\\pi"):
        """
        圓面積: pi * r^2
        例: circle_area(3) → "9\\pi"
        """
        # 回傳 LaTeX 格式
        return f"{radius**2}{pi_symbol}"
    
    @staticmethod
    def pythagorean(a, b):
        """
        勾股定理: c = sqrt(a^2 + b^2)
        """
        return RadicalOps.create(a**2 + b**2)
```

**關鍵點**:
- 使用 `@staticmethod` 保持一致的設計模式
- 充分利用已有的 `FractionOps`, `RadicalOps` 等
- 提供明確的文檔和範例
