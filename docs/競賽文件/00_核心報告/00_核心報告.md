### 【重要發現 3】Prompt 文檔矛盾與雙層防護

**發現時間**: 2026-02-03

**問題描述**:
- Prompt 文檔中「答案格式規範」出現 5 處不一致
- 導致 AI 生成答案時格式不統一（可能造成評測失敗）

**修復策略** (雙層防護):
```
【Layer 1 - 源頭修復】Prompt 里層
┌─ 統一答案格式定義（換行、空格分隔符）
├─ 移除模糊表述
└─ 確保 Gemini 和 Qwen 對齊

【Layer 2 - 事後修復】Healer Rule 2B
┌─ Regex 模式匹配 (空格轉換為 \n)
├─ AST 驗證（確保修復不破壞邏輯）
└─ 降級處理（無法修復時保留原文）
```

**驗證結果**: 9 個測試案例 100% 通過

## 💡 核心技術方案

### 方案 1：MCRI 評分系統

**定義**: 教育軟體可靠性綜合指標

| 層級 | 名稱 | 配分 | 評測內容 |
|------|------|------|---------|
| **L1** | 工程基石 | 20分 | 執行成功率、無 Crash |
| **L2** | 資料衛生 | 20分 | 無答案洩漏、格式正確 |
| **L3** | 評測公平 | 30分 | **外在強健性** ⭐（新） |
| **L4** | 教學有效 | 30分 | 題目符合課程目標 |

**創新點**: L3 外在強健性測試（首創）
- 測試代碼對「不同學生輸入」的容錯能力
- 不同於傳統代碼品質指標（只看源代碼）
- 真正評測「教學有效性」

### 方案 2：9 層 Healer Pipeline

```
Layer 1: Whitespace Normalization
  └─ 統一行尾、縮排、空白行

Layer 2: Import Removal
  └─ 移除危險庫 (exec、eval、__import__)

Layer 3: Output Wrapping
  └─ 確保 print() 在函數末尾

Layer 4: Anti-Override Protection
  └─ 不做無謂覆蓋（保留用戶邏輯）

Layer 5: Answer Space Fix (Rule 2B) ⭐
  └─ 空格轉 \n（答案格式修復）

Layer 6: Regex Validation Engine
  └─ 複雜模式匹配（LaTeX保護、字符編碼）

Layer 7: AST Static Analysis
  └─ 語法樹解析（邏輯驗證、類型檢查）

Layer 8: Loop Breaker
  └─ 偵測無窮迴圈（添加計數器）

Layer 9: Sandbox Execution + Fallback
  └─ 真實執行 + 失敗回退
```

**成效**: 從 15% (Ab1) → 98% (Ab3)
