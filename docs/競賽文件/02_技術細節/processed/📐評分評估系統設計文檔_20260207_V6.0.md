## 🎯 V6.0 系統核心目標
本系統基於 **MCRI V6.6 雙軌評分架構** (Program Value + Math Value)，旨在實現：
1. **全自動化評測**: 從 Golden Prompt 讀取到評分存儲的全流程。
2. **50/50 雙軌評價**: 同時關注程式工程品質與數學教學價值。
3. **深度質量控制**: 透過 L4.3/L4.4 指標過濾低質量數學題。
4. **架構品質分析**: 透過 L5 指標量化代碼的工程水準。

## 📊 V6.0 評分架構映射

新系統將評分結果映射到雙軌體系：

### **1. 程式價值 (Program Value, Max 50)**
用於評估 Agent 作為「工程師」的能力。
- **L1 工程基石** (15分): 執行穩定性、Docstring check (-2.0)、Performance (<1.0s)。
- **L2 資料衛生** (15分): 格式合規性、No Chinese (-2.5)、No Trailing Spaces。
- **L5 架構品質** (20分): AST 靜態分析 + Robustness (Retry/Healer)。

### **2. 數學價值 (Math Value, Max 50)**
用於評估 Agent 作為「數學老師」的能力。
- **L3 數學正確** (20分): 邏輯自洽 + 雙軌驗證 (SymPy + Check)。
- **L4 教學有效** (30分): 數值友善、LaTeX 規範、MQI 難度補償 (Bonus)。

## 📈 統計指標設計 (V6.0 新增)

### 雙軌分析
除了總分 (MCRI Total) 外，我們現在可以獨立分析 Model 的傾向：
- **High Program, Low Math**: "優秀的工程師，但數學不好" (常見於通用 Code 模型)
- **Low Program, High Math**: "數學天才，但寫不出好程式" (常見於數學專用模型)
- **Balanced**: "理想的 AI 助教" (Ab3 System Healer 的目標)

### MQI (Math Quality Index)
量化題目是否符合 K-12 教學規範的關鍵指標。
- **Perfect (10)**: 無任何冗餘係數或符號問題。
- **Acceptable (5-9)**: 輕微格式問題 (如 `1x`)。
- **Poor (<5)**: 嚴重錯誤 (如 `0x^2`, `+-`)。
