## 🎯 系統概述

### 背景與目標
當前系統已具備：
- ✅ Golden Prompt 讀取機制（模式 2）
- ✅ Golden Prompt 自動保存機制（模式 4）
- ✅ MCRI 評分系統（100 分制，四層級）
- ✅ Ablation Study 框架（Ab1/Ab2/Ab3）

**新系統目標**：
1. **批量生成**: 一次生成 1~5 個問題
2. **評分存儲**: 自動評分並入庫
3. **統計分析**: 生成統計表與統計圖
4. **實驗支持**: 支撐 2+1+15 混合實驗策略

## 📊 系統架構設計

### 核心模塊分解

```
golden_prompt_evaluation_system/
├── 1. GoldenPromptBatchGenerator
│   ├── 讀取 Golden Prompt 文件
│   ├── 配置生成參數（Ab1/Ab2/Ab3）

├── 4. StatisticsAnalyzer
│   ├── 聚合統計（平均分、標準差、分佈）

│   └── 趨勢分析（多次實驗累計）
│
└── 5. ReportGenerator
    ├── 統計表生成（CSV / HTML / Markdown）
    ├── 統計圖生成（matplotlib）
    └── 交互式報告（HTML + Chart.js）
```

## 📋 數據庫設計

### 新表: `golden_prompt_evaluations`

**用途**: 記錄 Golden Prompt 評分結果，支持統計分析

```sql
CREATE TABLE golden_prompt_evaluations (
    -- 主鍵與外鍵
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    experiment_run_id TEXT NOT NULL,  -- 本次實驗的唯一 ID
    skill_id TEXT NOT NULL,           -- 技能 ID (FK)
    ablation_type TEXT NOT NULL,      -- Ab1 / Ab2 / Ab3
    batch_idx INTEGER NOT NULL,       -- 批次索引 (0~4)
    
    -- 生成的代碼
    generated_code TEXT NOT NULL,     -- 完整代碼（用於復現）
    code_length INTEGER,              -- 代碼長度（統計用）
    
    -- MCRI 評分結果
    mcri_total_score REAL NOT NULL,   -- 總分 (0~100)
    mcri_l1_score REAL,               -- L1 分數
    mcri_l2_score REAL,               -- L2 分數
    mcri_l3_score REAL,               -- L3 分數
    mcri_l4_score REAL,               -- L4 分數
    
    -- 詳細評分信息（JSON）
    mcri_details TEXT,                -- MCRI 檢測結果詳情（JSON）
    
    -- 元數據
    model_name TEXT,                  -- 使用的 LLM 模型
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    notes TEXT,                       -- 備註欄
    
    -- 約束
    FOREIGN KEY (skill_id) REFERENCES skills_info(skill_id),
    UNIQUE (experiment_run_id, skill_id, ablation_type, batch_idx)
);

-- 索引優化查詢性能
CREATE INDEX idx_experiment_run ON golden_prompt_evaluations(experiment_run_id);
CREATE INDEX idx_skill_ablation ON golden_prompt_evaluations(skill_id, ablation_type);
CREATE INDEX idx_created_at ON golden_prompt_evaluations(created_at);
```

**欄位說明**:
- `experiment_run_id`: 識別同一次實驗運行（格式: `exp_20260205_001`）
- `batch_idx`: 在 1~5 個生成中的索引（0-based）
- `mcri_details`: JSON 格式存儲詳細評分（如各項檢測結果）

## 🛠️ 模塊設計細節

### 1️⃣ GoldenPromptBatchGenerator

**職責**: 讀取 Golden Prompt，批量生成問題

**輸入**:
```python
{
    'skill_id': 'jh_LinearEquations',
    'ablation_type': 'Ab3',           # Ab1/Ab2/Ab3
    'batch_size': 3,                  # 1~5
    'model_name': 'gemini-2.5-flash'  # 或其他模型
}
```

**輸出**:
```python
{
    'experiment_run_id': 'exp_20260205_001',
    'skill_id': 'jh_LinearEquations',
    'ablation_type': 'Ab3',
    'batch_size': 3,
    'generated_codes': [code_1, code_2, code_3],
    'generation_times': [2.3, 2.1, 2.4],  # 秒
    'validation_status': 'OK',        # PASS / FAIL
    'timestamp': '2026-02-05 10:30:00'
}
```

**核心邏輯**:
1. 讀取 Golden Prompt 文件（支持 3 種路徑）
2. 驗證 Prompt 有效性（非空、包含必要 markers）
3. 循環調用代碼生成器 `batch_size` 次
4. 基本質量檢查（AST 語法驗證，無需執行）
5. 返回結構化結果

**錯誤處理**:
- 文件不存在 → Fallback 動態生成（可配置）
- Prompt 無效 → 拋出異常，中止批次
- 生成失敗 → 記錄失敗原因，繼續下個批次

## 📈 統計指標設計

### 核心指標

| 指標 | 計算方式 | 用途 |
|------|--------|------|
| **平均分** | $\bar{x} = \frac{1}{n}\sum x_i$ | 整體品質 |
| **標準差** | $\sigma = \sqrt{\frac{1}{n}\sum(x_i - \bar{x})^2}$ | 穩定性 |
| **成功率** | $P(\text{score} > 80)$ | 可用率 |
| **95% CI** | $\bar{x} \pm 1.96 \frac{\sigma}{\sqrt{n}}$ | 置信區間 |
| **第 25/50/75 百分位** | 四分位數 | 分佈形狀 |

### 對比分析指標

| 指標 | 計算方式 | 解讀 |
|------|--------|------|
| **Δ Mean** | $\bar{x}_{Ab3} - \bar{x}_{Ab1}$ | Healer 的絕對貢獻 |
| **Δ Std** | $\sigma_{Ab1} - \sigma_{Ab3}$ | 穩定性改進 |
| **Δ Success** | $P(Ab3) - P(Ab1)$ | 可用率提升 |
| **Effect Size (Cohen's d)** | $\frac{\bar{x}_{Ab3} - \bar{x}_{Ab1}}{\sigma_{\text{pooled}}}$ | 標準化效果量 |

## 🔍 質量控制設計

### 驗證機制

| 階段 | 檢查項目 | 動作 |
|------|--------|------|
| **生成前** | Prompt 有效性 | 失敗則中止 |
| **生成後** | 代碼語法 | 失敗則標記，續續評分 |
| **評分後** | 分數有效性 | 異常則重評 |
| **存檔前** | 數據完整性 | 缺失則拒絕 |
| **統計前** | 樣本量 | 小於 3 則警告 |

### 錯誤恢復策略

- **Prompt 讀取失敗**: Fallback 動態生成
- **代碼生成失敗**: 標記並跳過，繼續下個批次
- **評分超時**: 返回 -1，標記異常
- **數據庫連接失敗**: 本地緩存，待連接恢復後同步
