## 🔑 关键设计决策

### 1. 为什么使用独立的 v01_pipeline.py？
- **原因**: 不修改原有的 code_generator.py（V9.2.0），避免破坏现有功能
- **优势**: 
  - v0.1 是并行的实验版本
  - 可以独立测试、验证、迭代
  - 如果成功，再逐步集成到主系统

### 2. Architect 为什么分析课本例题？
- **原因**: 遵循「Database-Driven Approach」，不硬编码 SPEC
- **流程**:
  ```
  課本例題（從 database）→ Architect 分析 → MASTER_SPEC → Coder 實現
  ```
- **优势**: 
  - 数据驱动，可扩展到其他 7 个技能
  - SPEC 自动提取，不需要人工定义

### 3. 为什么数值范围限制在 [-10, 10] 和 [-5, 5]？
- **系数范围 [-10, 10]**: 保证计算不会太复杂
  - 例: a=10, d=10 → f(5) = 10×125 + ... ≈ 1000+（可手算）
- **切点范围 [-5, 5]**: 进一步简化
  - 确保 f(x0) 结果在可接受范围
  - 切线斜率 m = f'(x0) 也不会过大
- **验证**: 3 次测试中所有数值都是「好计算、有意义」的
